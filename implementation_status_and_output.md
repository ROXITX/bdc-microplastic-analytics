# Project Implementation Status and Output Report

## 1. Implementation Status of Innovations

Based on the codebase (`pipeline.py`, `app.py`, `generate_actual_dataset.py`), here is the implementation status of the three major innovations and the interactive map component mentioned in your project document:

### 1. Multi-source Data Fusion (Implementation Status: **Partially Implemented / Simulated**)
* **Requirement**: Combining three different datasets (NOAA, Ocean Cleanup, Copernicus) into a single Big Data pipeline.
* **Current State**: The underlying logic for the features exists, but the pipeline currently reads a *single* combined dataset (`ocean_sensor_dataset.csv`). The script `generate_actual_dataset.py` creates this combined dataset by generating synthetic data for all the necessary features (sea surface temperature, wind speed, ocean currents, chlorophyll, etc.) into one CSV file. Real-time fusion of three distinct sources directly within `pipeline.py` is simulated rather than performed dynamically during the pipeline run.

### 2. Two-layer Machine Learning Architecture (Implementation Status: **Fully Implemented**)
* **Requirement**: Using KMeans clustering first, and using its output as an input feature for the RandomForest model.
* **Current State**: Implemented in `pipeline.py`. The pipeline fits a `KMeans(k=5)` model to generate a `cluster` column. This `cluster` column is then explicitly added to the feature vector (`rf_features = features + ["cluster"]`) used to train the `RandomForestClassifier`.

### 3. 30-day Rolling Window Feature (Implementation Status: **Fully Implemented**)
* **Requirement**: Capturing ocean current movement over time to make the model time-aware.
* **Current State**: Implemented in `pipeline.py` using PySpark's Window functions. It partitions data by Spatial Grid coordinates and orders by time to calculate the average ocean current velocity over the last 30 days (`rangeBetween(-days(30), 0)`).

### 4. Interactive Map for Decision Making (Implementation Status: **Fully Implemented**)
* **Requirement**: Creating an interactive map for use by environmental organizations.
* **Current State**: Implemented in `app.py` using **Streamlit**. The app reads the predictions generated by the pipeline (`dashboard_data/predictions.json`) and displays an interactive accumulation map (`st.map`), styling points differently based on whether they are predicted as a hotspot and visualizing the individual clusters. It also visually tracks model accuracy and silhouette scores.

## 3. How the System Works (End-to-End Workflow)

1. **Data Generation (`generate_actual_dataset.py`)**: 
   * This script simulates the initial data engineering phase. It creates 100,000 realistic records with features derived from NOAA (Temperature, Salinity) and Copernicus (Ocean Current, Chlorophyll). 
   * It specifically forces higher microplastic concentrations near 5 known ocean gyres (e.g., North Pacific, South Atlantic) to establish "Hotspots".

2. **Big Data Processing (`pipeline.py`)**:
   * **Ingestion**: PySpark reads the 100,000+ row dataset and handles nulls. It determines a dynamic threshold (top 25%) to define a `Hotspot`.
   * **Spatio-Temporal Features**: It rounds Lat/Lon coordinates to create geographical grid squares and uses PySpark Window functions to calculate the `Rolling_30D_Current` over time.
   * **Clustering Layer**: The data is scaled, and a KMeans algorithm identifies 5 distinct accumulation clusters in the data.
   * **Classification Layer**: The dataset (now containing the KMeans cluster identifier) is passed to a Random Forest Classifier to learn the interaction between environmental metrics and the likelihood of a grid cell becoming a Hotspot.
   * **Export**: A scaled-down sample (2000 points) of the predictions, along with the performance metrics, are exported to `dashboard_data/predictions.json` to keep frontend rendering smooth.

3. **Dashboard Web App (`app.py`)**:
   * Uses Streamlit to provide a web interface showcasing the metrics parsed from the JSON file. 
   * It renders a glassmorphism-styled UI containing KPIs (Model Accuracy, Quality).
   * It charts the Feature Importance (showing which environmental factors matter most) and renders the interactive map visualization of the hotspots. 

You can view the dashboard by keeping `streamlit run app.py` running in your terminal.
